# Список не проголосовавших в четверговом напоминании

Дата: 2026-07-22
Статус: утверждено

## Зачем

Напоминание о дедлайне (`scripts/telegram/notify.js`, режим `deadline`) сейчас шлёт голый текст
про дедлайн без данных о том, кто уже поставил прогноз. К четвергу — последнему дню перед
дедлайном — хочется мягко подтолкнуть тех, кто ещё не успел, показав их поимённо в самом чате.

## Что не меняется

- Расписание (`cron`) и `.github/workflows/telegram-notify.yml` — не трогаем. Режим `deadline`
  как срабатывал 4 раза в неделю (Ср 10:00, Ср 19:00, Чт 10:00, Чт 19:00 МСК), так и срабатывает.
- Режимы `raceweek`/`results`/`remind` — не трогаем.
- Текст среды не меняется.

## Что меняется

Только функция `deadline()` в `scripts/telegram/notify.js`.

### Определение четверга

День недели вычисляется по МСК внутри самого скрипта, без новых режимов и без изменений в
workflow:

```js
function isMskThursday() {
  return new Date().toLocaleString('en-US', { timeZone: 'Europe/Moscow', weekday: 'short' }) === 'Thu';
}
```

Если не четверг — сообщение как сейчас, без списка.

### Список не проголосовавших

Если четверг — для каждой гонки из `thisWeekOpenRaces()`:

1. `select user_id from predictions where race_id = $1` → множество проголосовавших `user_id`.
2. `select id, display_name from users` → полный ростер лиги (тот же запрос, что `listUsers()`
   на фронте, `src/lib/db.ts`).
3. Не проголосовавшие = ростер минус проголосовавшие, сортировка по `display_name`.

Если список пуст → строка `Все уже поставили прогноз, красавцы! 👍`.
Если не пуст → строка `Ещё не поставили: {Имя1}, {Имя2}, ...` (имена через запятую, без
Telegram-упоминаний/тегов — в чате нет привязки username к `display_name`).

Строка добавляется отдельным абзацем в конец существующего текста напоминания.

### Формат сообщения (четверг, пример)

```
⏰ Не забудь поставить прогноз на Belgian Grand Prix!
Дедлайн — четверг 19:00 МСК.
https://konicaru.github.io/f1-predict/predict

Ещё не поставили: Иван, Павел
```

В среду — без изменений (текущий текст, без последнего абзаца).

## Вне охвата

- Не меняем cron/workflow.
- Не трогаем `raceweek`/`results`/`remind`.
- Не тегаем людей персонально (нет маппинга username → display_name).
- Не различаем «10:00 четверга» и «19:00 четверга» — список показывается в оба четверговых запуска.

## Тестирование

Ручная проверка на реальных данных: прогнать `node scripts/telegram/notify.js deadline` локально
в четверг (или временно подменить `isMskThursday()`/системное время для проверки обеих веток —
четверг и не-четверг) с выводом в тестовый чат, сверить список не проголосовавших с фактическим
состоянием таблицы `predictions` для открытой гонки недели.
